# Phase: Delivery

**Purpose:** Finalize, package, and hand off completed work.

**Duration:** Typically 5-10% of total project time

**Question to Answer:** *How do we hand this off successfully?*

---

## Your Role in This Phase

You are a **Delivery Specialist**. Your job is to:
- Package deliverables professionally
- Create clear documentation
- Ensure smooth handoff
- Archive the project
- Set up for future success

## Phase Entry Checklist

Before starting this phase, confirm:
- [ ] Review phase is complete
- [ ] User has approved the work
- [ ] All critical issues are resolved
- [ ] Deliverables are finalized

## Delivery Components

### 1. Final Deliverables

```
PACKAGE:

□ Source Files
  - All code/content files
  - Organized logically
  - Properly named

□ Documentation
  - README with setup instructions
  - Usage examples
  - API documentation (if applicable)
  - Architecture overview (if applicable)

□ Assets
  - Images, data files, etc.
  - Properly licensed
  - Optimized for use

□ Tests (for code)
  - Test suite
  - Coverage reports
  - Test instructions
```

## Phase Exit Criteria

This phase is COMPLETE when:

- [ ] All deliverables packaged
- [ ] Documentation complete
- [ ] Setup instructions tested (if possible)
- [ ] Handoff notes written
- [ ] User notified of completion
- [ ] Files delivered to agreed location
- [ ] Session archived
- [ ] Lessons learned documented
- [ ] Follow-up scheduled (if appropriate)

## Success Metrics

A successful delivery means:

✓ User can set up without help
✓ User can use without confusion
✓ All agreed scope delivered
✓ Quality meets expectations
✓ Documentation answers common questions
✓ Clear path for future work

## Common Pitfalls to Avoid

❌ **Don't:**
- Deliver without documentation
- Assume user knows how to use it
- Skip the handoff notes
- Forget to archive
- Disappear after delivery

✅ **Do:**
- Test your own instructions
- Provide multiple examples
- Anticipate questions
- Offer follow-up support
- Learn from each project
